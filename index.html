Scroll Progress Bar

A pure CSS reading progress indicator using scroll-timeline. No JavaScript needed. The bar scales from 0 to 100% as you scroll the container.

Live Demo scroll-timeline

Scroll this container to see the progress bar animate. This uses the CSS scroll-timeline property to create a named timeline bound to the scroll position of this container.

The progress bar element uses animation-timeline: --main-scroll to link its scale transform to the scroll position. At 0% scroll, the bar is scaleX(0). At 100% scroll, it reaches scaleX(1).

This is a fundamental shift in how we think about scroll-linked animations. Previously, you'd need an Intersection Observer or a scroll event listener with requestAnimationFrame. Now it's pure CSS.

The scroll-timeline shorthand combines scroll-timeline-name and scroll-timeline-axis. The name must start with -- (like custom properties).

The animation itself is a standard @keyframes rule. The only difference is that instead of running over a time duration, it runs over the scroll distance. The linear easing ensures smooth 1:1 mapping.

You can use any animatable CSS property here: opacity, transforms, colors, clip-path, filters. The scroll position becomes your animation controller.

Performance is excellent because scroll-timeline animations run on the compositor thread, just like CSS transitions. They don't cause layout recalculations or trigger JavaScript.

One important detail: the element with scroll-timeline must be a scroll container (have overflow: auto or scroll). The animated element must be able to find the named timeline in its ancestor chain.

Try combining this with animation-range to control when the animation starts and ends within the scroll range. For example, you could make the bar only animate in the first 50% of scroll.

This pattern works great for article reading indicators, section progress trackers, horizontal scrolling galleries, and any UI where visual feedback tied to scroll position is valuable.

.container { overflow-y: auto; scroll-timeline: --main-scroll y; } .progress-bar { position: sticky; top: 0; height: 4px; background: #3b82f6; transform-origin: left; transform: scaleX(0); animation: grow linear; animation-timeline: --main-scroll; } @keyframes grow { to { transform: scaleX(1); } }

How It Works

1. Create a scroll timeline
Apply scroll-timeline: --name axis to a scrollable container. This registers a named timeline scoped to that element's scroll range.

2. Link an animation
On the element you want to animate, use animation-timeline: --name to bind a @keyframes animation to the scroll progress instead of time.

3. Define keyframes
Write standard @keyframes. The browser maps 0% to scroll-start and 100% to scroll-end. Use linear easing for direct 1:1 mapping.

4. Compositor-driven
These animations run on the compositor thread, making them buttery smooth with zero main-thread overhead. No JavaScript. No jank.

Key Properties:

  • scroll-timeline-name — names the timeline
  • scroll-timeline-axis — y (default), x, block, inline
  • animation-timeline — links to named or anonymous timeline
  • scroll() — anonymous scroll timeline function

Parallax Effects

Create depth and motion with scroll-driven parallax layers. Different elements move at different speeds, creating an immersive scrolling experience — all in CSS.

Multi-Layer Parallax scroll-timeline

Layer 1: Background

The gradient background moves at a different rate than the content, creating a parallax depth effect. This uses a separate @keyframes with translateY bound to the same scroll-timeline.

Layer 2: Content Cards

Each card fades in and out as it enters and exits the viewport. The animation-range property controls exactly when each card's animation begins and ends within the scroll range.

Layer 3: Staggered Timing

By adjusting animation-range values on each card, we create a staggered reveal effect. Earlier cards animate first, later cards animate as you continue scrolling.

Layer 4: Composited

All these animations are GPU-accelerated and composited. No layout thrashing, no JavaScript scroll handlers, no requestAnimationFrame. Pure CSS scroll-driven animations.

.container { scroll-timeline: --parallax y; } .background { animation: parallaxBg linear; animation-timeline: --parallax; } .card { animation: fadeSlide linear both; animation-timeline: --parallax; animation-range: 20% 50%; /* per card */ } @keyframes parallaxBg { to { transform: translateY(-100px); } } @keyframes fadeSlide { 0% { opacity: 0; transform: translateY(40px); } 25% { opacity: 1; transform: translateY(0); } 75% { opacity: 1; transform: translateY(0); } 100% { opacity: 0; transform: translateY(-40px); } }
